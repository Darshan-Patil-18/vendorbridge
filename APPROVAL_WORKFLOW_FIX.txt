PROBLEM:
--------
The Approval Workflow page was showing "Error loading approvals" with a 400 
status error. The issue was caused by:

1. Foreign key constraint violations when querying the approvals table
2. Complex JOIN operations trying to reference profiles table with incorrect FK
3. The query was using: profiles!approvals_requested_by_fkey(full_name)
4. This foreign key reference doesn't exist or is misconfigured

ERROR MESSAGES IN CONSOLE:
--------------------------
- "Failed to load resource: the server responded with a status of 400"
- Foreign key violation errors
- "Could not find a foreign key relationship between approvals and profiles"

SOLUTION APPLIED:
-----------------
Complete rewrite of the fetchApprovals() function to:

1. ✅ Fetch approvals table FIRST without any JOINs
2. ✅ Extract unique IDs (rfq_id, quotation_id, requested_by)
3. ✅ Fetch related tables in PARALLEL using Promise.all()
4. ✅ Create lookup maps for efficient data joining
5. ✅ Manually map all data together
6. ✅ NO FOREIGN KEY DEPENDENCIES
7. ✅ Extensive error handling and console logging
8. ✅ Graceful fallbacks for missing data

KEY CHANGES:
------------
Before: Complex single query with nested JOINs and FK references
After: Multiple simple queries with manual data mapping

Before:
```javascript
.select(`
  *,
  rfqs(id, title, priority, created_at),
  quotations(id, vendor_name, total_amount, vendor_id),
  profiles!approvals_requested_by_fkey(full_name)
`)
```

After:
```javascript
// Step 1: Fetch approvals alone
const { data: approvalsData } = await supabase.from('approvals').select('*')

// Step 2: Fetch related tables separately
await Promise.all([
  supabase.from('rfqs').select(...).in('id', rfqIds),
  supabase.from('quotations').select(...).in('id', quotationIds),
  supabase.from('profiles').select(...).in('id', userIds)
])

// Step 3: Map data manually
```
